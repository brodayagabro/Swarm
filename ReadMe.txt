В проекте реализовано три класса в следующих зависимостях:

Robot->WorkingRobot->CommanderRobot

Реализованы методы, описанные в задании, перегрузки операторов.

Есть закомментированные куски кода, в которых пока непонятно, что должно происходидить, в основном это связано с кэтчем исключений в тестах.


Организация проекта

./src -  файлы с определениями методов классов и тестов *.cpp

./obj - в эту дирректорию генерируются объектные модули *.o

./gtestd - здесь определны файлы с тестами классов.
 Можете пожалуйста почитать. Я организовал тест каждого класса в отдельном заголовочном файле. Не уверен корректно ли это писать в рамках заголовочника.

./hdr -заголовочные файлы

./deps файлы зависимостей

Конечный файл с тестами ggtest*

для сборки под Линукс - make
